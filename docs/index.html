Core idea

Every issue is triaged before any action

Risk classification decides whether a fix is applied automatically, proposed for approval, or flagged for a human.

● Low risk

Auto-fixed

Deterministic, structural, no content judgment needed — fixed automatically and logged.

  • Missing slide titles
  • Generic link text ("click here")
  • Missing language metadata
  • Duplicate slide titles
● Medium risk

Suggested → you approve

Needs context, but a good fix can be drafted. Staged as pending_approval until a human decides.

  • Missing / auto-generated alt text
  • Filename-like alt text
  • Inferred reading order
  • Contrast near a compliant color
● High risk

Flagged for a human

Requires understanding of meaning and intent. Described in plain language with WCAG guidance — no auto edit.

  • Complex charts & diagrams
  • Color-as-only-differentiator
  • Meaning-sensitive text
  • Nested tables, embedded media
What's inside

Built for trust and accountability

Nothing changes silently. Every fix is snapshotted, logged, and reversible.

🔍

Rule-based detection

One module per WCAG / Section 508 check, returning structured findings mapped to specific criteria.

⚡

Safe auto-fix

Low-risk issues repaired atomically — placeholder titles, normalized links, injected language metadata.

🤝

Human-in-the-loop review

Approve, edit, or reject each medium-risk suggestion — right in your browser (or the terminal). Your wording wins.

📒

Itemized audit ledger

Every detected issue produces one append-only JSON entry with criterion, risk, status, and timestamps.

↩️

Per-item rollback

Any applied fix reverts independently from its before/after snapshot — no chained dependencies.

📊

Exportable reports

Generate an XLSX audit spreadsheet (one row per item) and a PDF summary for stakeholders.

Step by step

Install it on your computer

No coding experience needed. Do this once, then it's a double-click every time. Prefer to watch? Follow the video for your system, then use the written steps below as a reference.

1

Download the app from GitHub

Open the project page at github.com/zhuohui-chen/AccessiSlides. Click the green <> Code button near the top-right, then choose Download ZIP at the bottom of the menu.

🍎 macOS

Find AccessiSlides-main.zip in your Downloads folder and double-click it. macOS unzips it into a folder named AccessiSlides-main.

🪟 Windows

In Downloads, right-click AccessiSlides-main.zip → Extract All… → Extract. You'll get an AccessiSlides-main folder.

💡 Tip: Move that folder somewhere easy to find, like your Desktop.
2

Install uv — the engine that runs everything

uv is a small, free tool that sets up Python and every other piece for you, so you don't have to install them one by one. You only do this once. Copy the line for your system, paste it into the terminal, and press Enter.

🍎 macOS

Open Terminal (press ⌘ Space, type “Terminal”, Enter), then paste:

curl -LsSf https://astral.sh/uv/install.sh | sh
🪟 Windows

Open PowerShell (Start menu → type “PowerShell”, Enter), then paste:

powershell -c "irm https://astral.sh/uv/install.ps1 | iex"
💡 When it finishes, close the terminal and open a new one so the uv command is recognized. Python is installed automatically by uv — you don't need to install it separately.
3

Point the terminal at the folder & set up

The terminal needs to be “inside” the AccessiSlides-main folder from Step 1:

🍎 macOS

In Terminal, type cd and a space, then drag the folder onto the window and press Enter.

🪟 Windows

Shift + right-click the folder → Open PowerShell window here (or “Open in Terminal”).

Then set everything up — this downloads Python and the libraries. The first run takes a minute:

uv sync
4

Launch the app

Open the AccessiSlides-main folder and double-click the launcher:

🍎 macOS

Start-AccessiSlides.command

🪟 Windows

Start-AccessiSlides.bat

Your browser opens automatically at http://127.0.0.1:8765. Prefer the terminal? Run uv run python cli.py serve instead.

🔐 First time only: on macOS you may see “cannot be opened because it is from an unidentified developer” — right-click the launcher → Open → Open. On Windows, if you see “Windows protected your PC”, click More info → Run anyway.
5

(Optional) Turn on AI suggestions

The app works great on its own with built-in fixes. For smarter alt-text and titles, open ⚙️ Settings in the app and paste an API key from Anthropic or OpenAI. The key is held in memory for that session only — never saved to disk or sent anywhere except your chosen AI provider.

6

Everyday use, stopping & updating

Use it again anytime: just double-click the launcher — no setup needed after the first time.

Stop the app: close the small terminal window that opened alongside it (or press Ctrl + C in it).

Update to a newer version: download the ZIP again (Step 1), replace the old folder, then run uv sync once more.

Command reference

Every command in one place

Already comfortable in a terminal? Here's the whole thing at a glance — setup, launching the app, and the full CLI. See the step-by-step install above if you'd rather be walked through it.

Clone the repo →
terminal 
# 1. Install uv (once per machine)
curl -LsSf https://astral.sh/uv/install.sh | sh

# 2. Set up the environment
uv sync

# 3. Launch the app → opens http://127.0.0.1:8765
#    (or just double-click Start-AccessiSlides.command / .bat)
uv run python cli.py serve

# ── Prefer the command line? Same engine, no browser: ──
uv run python cli.py check  --input slides.pptx
uv run python cli.py fix    --input slides.pptx --output fixed.pptx --ledger audit/ledger.json
uv run python cli.py report --ledger audit/ledger.json --pptx fixed.pptx --review
uv run python cli.py export --ledger audit/ledger.json --format pdf --output summary.pdf
